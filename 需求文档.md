# 金融计算器 & 股票回测平台 - 需求文档

**版本**: v1.2
**日期**: 2026-03-02

---

## 1. 项目概述

### 1.1 项目目标
做一个投资工具网站，帮助散户投资者：
1. **计算投资收益** - 各种投资相关的计算器
2. **模拟股票回测** - 用 A股历史数据测试投资策略

### 1.2 盈利模式
- 网站广告（Google AdSense 等）
- 预留会员功能接口（未来实现）

### 1.3 数据来源

#### 实时行情 + 历史数据
- **数据源**：腾讯财经 API（免费，无需 token）
- **更新方式**：每日自动抓取最新数据
- **存储方式**：本地 JSON 文件

#### 数据文件结构
```
project/
├── data/
│   ├── stocks.json        # 股票基本信息
│   ├── sh000001.json     # 上证指数历史K线
│   ├── sz000001.json     # 深证成指历史K线
│   └── ...
```

#### 数据更新流程
```
运行 data/update.js 脚本
    ↓
从腾讯财经API获取全市场股票列表
    ↓
获取每只股票的历史K线数据
    ↓
保存为JSON文件
    ↓
前端直接读取本地JSON（无需API调用）
```

## 2. 功能需求

### 2.1 投资计算器工具箱

以下计算器均在浏览器端执行，无需服务器：

#### 2.1.1 定投计算器
**功能**：计算定期投资基金的收益

**用户输入**：
- 每月投入金额（如：1000元）
- 预计投资年限（如：10年）
- 预计年化收益率（如：8%）

**输出结果**：
- 总投入本金
- 最终账户价值
- 总收益金额
- 总收益率

**展示方式**：
- 数字结果展示
- 收益曲线图（使用 Chart.js）

---

#### 2.1.2 复利计算器
**功能**：计算复利增长效果

**用户输入**：
- 初始本金（如：10万元）
- 年化收益率（如：10%）
- 投资年限（如：20年）

**输出结果**：
- 最终本息总额
- 收益明细表（每年年末金额）

**展示方式**：
- 数字结果
- 复利增长曲线图

---

#### 2.1.3 股票收益计算器
**功能**：计算买卖股票的盈亏

**用户输入**：
- 买入价格（如：10.5元）
- 卖出价格（如：12.3元）
- 买入数量（如：1000股）
- 手续费率（默认万3，可修改）
- 印花税率（默认千1，卖出时，可修改）

**输出结果**：
- 买入成本
- 卖出收入
- 手续费合计
- 印花税
- 净利润
- 收益率

---

#### 2.1.4 PE 估值计算器
**功能**：计算股票合理价格

**用户输入**：
- 每股收益 EPS
- 行业平均 PE
- 或者自定义目标 PE

**输出结果**：
- 合理估值价格区间

---

### 2.2 股票回测练习系统

#### 2.2.1 功能目标
让用户用 A股历史数据测试自己的投资策略，看看如果过去买了某只股票，现在能赚多少。

#### 2.2.2 核心功能

**A. 股票选择**
- 用户输入股票代码（如：000001）或名称（如：平安银行）
- 系统自动匹配并显示候选列表（从本地数据读取）
- 用户选择回测时间范围（开始日期 ~ 结束日期）
- 显示该股票的基本信息（最新价、涨跌幅等）

**B. 策略选择**

用户提供策略参数或选择预设策略：

1. **均线策略**
   - 短期均线上穿长期均线 = 买入
   - 短期均线下穿长期均线 = 卖出
   - 用户可设置：短期天数（如5日）、长期天数（如20日）

2. **金叉死叉策略**（同均线策略）

3. **定投策略**
   - 每月固定日期买入固定金额
   - 计算最终收益

4. **持有策略**
   - 买入后一直持有到结束
   - 计算总收益

5. **自定义策略**（未来版本）

**C. 回测执行**
- 点击"开始回测"按钮
- 系统根据选择的策略模拟交易
- 显示模拟的交易记录（买入日期、价格、数量）

**D. 回测结果展示**
- 总收益率
- 年化收益率
- 买入次数
- 卖出次数
- 胜率
- 收益曲线图（资金变化）
- 持仓变化图

## 3. 界面设计要求

- **电脑端**（> 1024px）：多列布局，完整展示

- 简洁、专业、清爽

- **深色**：深色背景 + 浅色文字
- **浅色**：浅色背景 + 深色文字
- **涨跌色**：根据用户设置变化

### 3.4 交互体验
- 输入框有明确的提示文字
- 计算/回测按钮明显易找

## 4. 技术方案

### 4.1 技术栈

#### 前端
- **框架**：原生 HTML + CSS + JavaScript
- **图表库**：Chart.js（用于绘制收益曲线）
- **数据存储**：
  - 股票数据：本地 JSON 文件（通过 fetch 加载）
  - 用户设置：浏览器 localStorage

#### 后端（数据更新脚本）
- **运行环境**：Node.js
- **数据源**：腾讯财经 API
- **输出**：JSON 文件

### 4.2 项目结构

```
project/
├── index.html              # 首页
├── calculator.html         # 计算器页面
├── backtest.html           # 回测页面
├── settings.html           # 设置页面
├── about.html              # 关于页面
├── css/
│   └── style.css          # 样式文件
├── js/
│   ├── main.js            # 主逻辑
│   ├── calculator.js      # 计算器逻辑
│   ├── backtest.js        # 回测逻辑
│   ├── settings.js        # 设置逻辑
│   └── theme.js           # 主题管理
├── data/
│   ├── stocks.json        # 股票列表
│   ├── sh000001.json     # 个股历史数据

```

### 4.3 数据更新流程

```
每日运行: node scripts/update-data.js
    ↓
获取A股股票列表（从腾讯API）
    ↓
遍历每只股票，获取历史K线
    ↓
保存为 data/xxxxxx.json
    ↓
完成
```

**首次运行**：需要抓取全部股票历史数据（耗时较长）
**日常运行**：只抓取最新一天数据（快速）

### 4.4 腾讯财经 API

#### 获取股票列表
```
https://push2.eastmoney.com/api/qt/clist/get?pn=1&pz=5000&po=1&np=1&ut=bd1d9ddb04089700cf9c27f6f7426281&fltt=2&invt=2&fid=f3&fs=m:0+t:6,m:0+t:80,m:1+t:2,m:1+t:23
```

#### 获取K线数据
```
https://web.ifzq.gtimg.cn/appstock/app/fqkline/get?param={code},day,,,{days},qfq
```
